# Routes Instructions

These instructions apply to files inside `routes/`.

## Goal

- Keep route files small, clear, and focused on endpoint wiring.
- Keep business logic out of route declarations.

## Route Rules

- Define routes, middleware chaining, and controller bindings in routes files.
- Do not place heavy business logic directly in route handlers when it can live in controllers or utilities.
- If route files become large, split by feature or resource into smaller route modules.
- Keep naming and URL structure consistent with existing route patterns.
- When splitting route modules, keep them grouped by endpoint family so the router tree stays easy for humans to follow at a glance.

## Structure Rules

- Use controllers for request handling and data flow.
- Keep repeated middleware composition reusable where possible.

## Validation

- Verify affected endpoints still resolve to the intended controller actions.
- Perform a small manual check for any modified route path.
