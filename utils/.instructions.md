# Utils Instructions

These instructions apply to files inside `utils/`.

## Goal

- Keep utility modules small, reusable, and easy to test.
- Centralize shared transformation and validation logic.

## Utility Rules

- Move repeated controller or model helper logic into utility modules when reuse is likely.
- Prefer pure functions with explicit inputs and outputs.
- Keep one utility file focused on a single concern.
- If a utility file grows large, split it into smaller concern-based modules.
- When splitting utilities, keep related transformations together, avoid generic grab-bags, and choose names that make the purpose obvious without reading the implementation.

## Design Rules

- Avoid hidden side effects unless clearly required by the use case.
- Keep function names descriptive and consistent with existing naming patterns.
- Reuse existing utility modules before creating new ones.

## Validation

- Validate utility behavior for edge cases related to touched functions.
- Run the smallest useful tests for code paths that depend on updated utilities.
